# Student Portal Development

**WORKFLOW SKILL** — Специализированный навык для создания интерфейсов студенческого портала в системах управления университетом, включая просмотр курсов, расписания, оценок, посещаемости и заданий.

WHEN: разработка студенческих дашбордов, порталов для студентов, интерфейсов для просмотра академической информации, приложений для отслеживания прогресса.

DO NOT USE FOR: общая разработка фронтенда (используйте основной агент); админские панели (используйте university-management-app); неакадемические порталы.

## Workflow Steps

1. **Анализ требований**
   - Определить роли пользователей (студент: только просмотр)
   - Выбрать необходимые функции: курсы, расписание, оценки, посещаемость, задания

2. **Дизайн интерфейса**
   - Создать макеты страниц: дашборд, курсы, расписание, оценки
   - Обеспечить responsive дизайн для мобильных устройств

3. **Разработка компонентов**
   - Реализовать компоненты для отображения данных
   - Добавить навигацию и меню
   - Интегрировать с API бэкенда

4. **Безопасность и валидация**
   - Реализовать аутентификацию JWT
   - Валидировать доступ только к данным студента
   - Обработать ошибки и загрузку

5. **Тестирование и оптимизация**
   - Протестировать все функции
   - Оптимизировать производительность
   - Подготовить к интеграции с основной системой

## Decision Points
- Компоненты: использовать React hooks или классы
- Стилизация: CSS modules или styled-components
- API интеграция: fetch или axios

## Quality Criteria
- Четкий и интуитивный интерфейс
- Быстрая загрузка данных
- Безопасный доступ к данным
- Совместимость с мобильными устройствами

INVOKES: file system tools для генерации кода, terminal для запуска фронтенда; ask-questions для уточнения требований.
